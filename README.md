# 🧠 RAG-агент для поиска мишеней при болезни Альцгеймера

Прототип RAG-агента (Retrieval-Augmented Generation), помогающего исследователям находить потенциальные мишени для разработки лекарств от болезни Альцгеймера на основе открытых научных публикаций.

---

## 📋 Описание

Система извлекает релевантeные фрагменты из корпуса научных статей и использует LLM-агента для генерации обоснованных ответов со ссылками на источники. Агент разграничивает **валидированные**, **кандидатные** и **спекулятивные** мишени.

### Примеры вопросов, на которые отвечает система

- _What are potential targets for Alzheimer's disease treatment?_
- _Are the targets druggable with small molecules, biologics, or other modalities?_
- _What additional studies are needed to advance these targets?_

## ⚙️ Запуск

### 1. Клонировать репозиторий

```bash
git clone https://github.com/<ваш-username>/alzheimer-rag.git
cd alzheimer-rag
```

### 2. Установить зависимости

```bash
pip install -r requirements.txt
```

### 3. Собрать данные

Запустить парсер для загрузки ~50 статей в открытом доступе из EuropePMC:

```bash
python parser.py
```

В директории появятся файлы `alzheimer_targets.json` и `alzheimer_targets.csv`.  
Количество статей регулируется параметром `MAX_PAPERS` в `parser.py` (от 10 до 100).

### 4. Запустить приложение

```bash
streamlit run app.py
```

| Шаг               | Инструмент                            | Описание                                             |
| ----------------- | ------------------------------------- | ---------------------------------------------------- |
| Поиск             | EuropePMC REST API                    | Запрос: Alzheimer + therapeutic target + open access |

| Генерация         | Qwen/Qwen3-14B через HuggingFace Hub  | 4-шаговый CodeAgent                                  |

## 📊 Оценка качества ретривера

Оценка проводилась вручную на 3 типичных исследовательских запросах по релевантности возвращаемых фрагментов.

|Метрика|Значение|
|---|---|
|Релевантных фрагментов среди всех полученных|25 / 30 (83%)|
|Нерелевантных фрагментов, использованных в ответе|0 / 5 (0%)|
|Пропущенных агентом релевантных фрагментов|1 / 25 (4%)|

**Вывод:** BM25-ретривер извлекает релевантный контент с высокой точностью. Агент не использует нерелевантные фрагменты и не галлюцинирует. Покрытие полезных фрагментов составляет 96%, что говорит о хорошей заземлённости генерации на источниках.

## ❓ Вопросы и ответы

### На какие модальности данных можно расширить решение?

Текущее решение работает только с текстом (полные тексты статей). Возможные расширения:

- **Структурированные данные** — базы белков (UniProt), базы взаимодействий лекарство-мишень (ChEMBL, DrugBank), реестры клинических испытаний (ClinicalTrials.gov). Позволяют отвечать на вопросы о druggability напрямую, без извлечения из текста.
- **Графы знаний** — сущности и отношения, извлечённые из статей, хранящиеся в графовой БД (Neo4j). Открывают многошаговые запросы: _ген → путь → препарат → испытание_.
- **Молекулярные данные** — структуры белков (PDB / AlphaFold).
- **Изображения и рисунки** — молекулярные диаграммы, схемы путей, микроскопия из статей (мультимодальные модели: InternVL, LLaVA-Med).
- **Омикс-данные** — экспрессия генов (GEO), протеомика, результаты GWAS. Статистические сигналы становятся ключами для поиска.

### Как это реализовать?

Для каждой модальности создаётся отдельный инструмент, который регистрируется у агента:

```python
agent = CodeAgent(
    tools=[
        text_retriever_tool,      # BM25 по текстам статей
        graph_retriever_tool,     # Cypher-запросы к Neo4j
        structure_retriever_tool, # Поиск по эмбеддингам PDB
        omics_retriever_tool,     # SQL / pandas по таблицам экспрессии
    ],
    model=...,
)
```

Агент самостоятельно решает, какой инструмент вызвать для конкретного запроса — в этом и состоит ключевое преимущество агентного подхода (CodeAgent) перед фиксированным RAG-пайплайном.

### Какие модели выбраны и почему?

| Компонент          | Выбор                              | Обоснование                                                                                                                                  |
| ------------------ | ---------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| LLM                | **Qwen/Qwen3-14B**                 | Сильное рассуждение, длинный контекст, открытый доступ на HuggingFace; хорошо следует инструкциям по форматированию цитат                    |
| Retriever          | **BM25**                           | Интерпретируемый, работает без обучения; хорошо подходит для доменной терминологии (названия генов, препаратов), где важно точное совпадение |
| Разбивка           | **RecursiveCharacterTextSplitter** | Сохраняет границы предложений; чанки по 1500 символов балансируют между полнотой контекста и точностью поиска                                |
| Агентный фреймворк | **smolagents CodeAgent**           | Генерирует Python-код для оркестрации вызовов инструментов — гибче, чем ReAct, для многошагового ретривера                                   |

Естественным следующим шагом стало бы добавление **dense retriever** (например, `BAAI/bge-m3` или биомедицинской модели `BioLORD`) в гибридной связке с BM25 через Reciprocal Rank Fusion — это улучшает recall на парафразированных научных запросах.
